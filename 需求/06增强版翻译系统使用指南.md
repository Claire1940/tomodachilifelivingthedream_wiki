# 增强版翻译系统使用指南

## 概述

增强版翻译系统解决了原翻译脚本的所有已知问题：
- ✅ Token 限制问题（自动分块）
- ✅ JSON 格式错误（自动验证）
- ✅ 字段错位问题（结构验证）
- ✅ 专有名词被翻译（自动保护）
- ✅ 翻译失败无提示（详细报告）

## 快速开始

### 基础翻译
```bash
# 翻译单个语言
python3 tools/articles/modules/transpage/translate-messages-enhanced.py --lang vi

# 翻译多个语言
python3 tools/articles/modules/transpage/translate-messages-enhanced.py --lang vi,th,es

# 强制覆盖已有翻译
python3 tools/articles/modules/transpage/translate-messages-enhanced.py --lang vi --overwrite
```

### 增量翻译（推荐）
```bash
# 只翻译新增或修改的内容
python3 tools/articles/modules/transpage/translate-messages-enhanced.py --lang vi --incremental

# 增量翻译 + 生成报告
python3 tools/articles/modules/transpage/translate-messages-enhanced.py --lang vi --incremental --report
```

### 指定分块策略
```bash
# 使用中等分块（30 个键一块）
python3 tools/articles/modules/transpage/translate-messages-enhanced.py --lang vi --strategy medium

# 使用小分块（15 个键一块）
python3 tools/articles/modules/transpage/translate-messages-enhanced.py --lang vi --strategy small
```

## 核心特性

### 1. 智能分块翻译

**问题**：原脚本遇到 token 限制（8,192 tokens）导致翻译截断

**解决方案**：自动分块 + 降级重试

```
策略 1: top_level（按顶层键分块）
  ├─ common
  ├─ hero
  ├─ features
  └─ ...

如果失败 ↓

策略 2: medium（30 个键一块）
  ├─ chunk_00 (30 keys)
  ├─ chunk_01 (30 keys)
  └─ ...

如果失败 ↓

策略 3: small（15 个键一块）
  └─ ...

如果失败 ↓

策略 4: tiny（5 个键一块）
  └─ 最后手段
```

### 2. 翻译结果验证

每个翻译结果都会经过三重验证：

```python
✓ JSON 格式验证
  - 检查 JSON 是否有效
  - 检查括号、引号是否匹配

✓ 结构完整性验证
  - 检查所有键是否存在
  - 检查嵌套层级是否正确

✓ 文件大小验证
  - 检查翻译后大小是否合理（±50%）
  - 防止翻译丢失或重复
```

### 3. 专有名词保护

**配置文件**：`tools/articles/modules/transpage/transpage_config.json`

```json
{
  "protected_terms": {
    "game_names": ["Burglin' Gnomes"],
    "character_names": ["Jotaro", "DIO", "Kakyoin"],
    "technical_terms": ["PvP", "PvE", "HP", "XP", "DPS"]
  }
}
```

**工作原理**：
1. 翻译前：将专有名词替换为占位符 `__PROTECTED_abc123__`
2. 翻译中：AI 不会翻译占位符
3. 翻译后：将占位符还原为原始名词

### 4. 增量翻译

**使用场景**：en.json 有小幅更新，不想重新翻译整个文件

**工作流程**：
```bash
# 1. 备份当前 en.json（可选）
cp src/locales/en.json src/locales/en.json.backup

# 2. 修改 en.json（添加新字段或修改现有字段）

# 3. 运行增量翻译
python3 tools/articles/modules/transpage/translate-messages-enhanced.py --lang vi --incremental
```

**效果**：
- 只翻译变化的顶层键（如只翻译 `hero` 部分）
- 保留其他部分的现有翻译
- 大幅节省时间和 API 成本

## 常见问题

### Q1: 翻译失败怎么办？

**A**: 增强版脚本会自动降级重试，如果所有策略都失败：

1. 查看错误报告（如果启用了 `--report`）
2. 尝试手动指定更小的分块策略：
   ```bash
   python3 translate-messages-enhanced.py --lang vi --strategy small
   ```
3. 如果仍然失败，使用最小分块：
   ```bash
   python3 translate-messages-enhanced.py --lang vi --strategy tiny
   ```

### Q2: 如何添加新的专有名词？

**A**: 编辑 `transpage_config.json`：

```json
{
  "protected_terms": {
    "game_names": ["Burglin' Gnomes", "Your Game Name"],
    "character_names": ["Hero1", "Hero2"],
    "technical_terms": ["PvP", "CustomTerm"]
  }
}
```

### Q3: 增量翻译如何判断哪些内容变化了？

**A**:
- 如果存在 `en.json.backup`，会对比备份和当前版本
- 如果没有备份，会翻译所有内容（等同于完整翻译）
- 建议在修改 en.json 前先备份

### Q4: 翻译后 JSON 格式错误怎么办？

**A**: 增强版脚本会自动验证 JSON 格式，如果验证失败会自动重试。如果仍然出错：

1. 查看错误信息（脚本会显示具体错误位置）
2. 使用 VS Code 打开文件，查看语法错误提示
3. 手动修复或重新翻译该语言

### Q5: 如何验证翻译质量？

**A**:
```bash
# 1. 验证 JSON 格式
python3 -m json.tool src/locales/vi.json > /dev/null && echo "Valid" || echo "Invalid"

# 2. 检查文件大小
ls -lh src/locales/en.json src/locales/vi.json

# 3. 在浏览器中测试
bun dev
# 切换到对应语言，检查显示是否正常
```

## 故障排除

### 问题 1: 所有策略都失败

**症状**：脚本显示 "所有分块策略都失败"

**解决方案**：
1. 检查 API 配置是否正确
2. 检查网络连接
3. 查看详细报告（`--report`）
4. 尝试减少并发语言数量

### 问题 2: 翻译后专有名词被翻译了

**症状**：游戏名称或角色名被翻译成目标语言

**解决方案**：
1. 检查 `transpage_config.json` 中的 `protected_terms` 配置
2. 确保专有名词拼写正确（区分大小写）
3. 重新翻译：
   ```bash
   python3 translate-messages-enhanced.py --lang vi --overwrite
   ```

### 问题 3: 增量翻译没有检测到变化

**症状**：脚本显示 "No changes detected"

**解决方案**：
1. 确保 `en.json.backup` 存在且是旧版本
2. 如果没有备份，使用完整翻译：
   ```bash
   python3 translate-messages-enhanced.py --lang vi --overwrite
   ```
